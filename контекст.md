# 🚀 Cartoon Agent --- Production Architecture (Расширенная версия)

------------------------------------------------------------------------

# 1. Общая архитектура системы

## 1.1 Компонентная диаграмма

``` mermaid
flowchart TD
    U[Пользователь] --> R[Command Router]
    R -->|Детерминированная команда| T[Tool Execution]
    R -->|Сложная задача| P[Planner LLM]

    P --> E[Execution Engine]
    T --> E

    E --> G[Git Layer]
    G --> V[Verification Layer]
    V --> UI[Trace / UI]

    E --> A[Asset Pipeline]
```

------------------------------------------------------------------------

## 1.2 Последовательность выполнения (Sequence Diagram)

``` mermaid
sequenceDiagram
    participant User
    participant Router
    participant Planner
    participant Engine
    participant Git
    participant Verify

    User->>Router: Команда
    Router->>Planner: Запрос JSON-плана
    Planner-->>Router: План (JSON)
    Router->>Engine: Выполнить действия
    Engine->>Git: Создать ветку / применить патч
    Engine->>Verify: Проверка
    Verify-->>Engine: OK / FAIL
    Engine-->>User: Результат + Trace + Diff
```

------------------------------------------------------------------------

# 2. Блок-схема self‑modification loop

``` mermaid
flowchart TD
    A[Получена ошибка] --> B[Сбор контекста]
    B --> C[LLM генерирует patch JSON]
    C --> D[Создание git-ветки]
    D --> E[Применение патча]
    E --> F[Verification]
    F -->|Ошибка| B
    F -->|OK| G[Показ diff пользователю]
    G --> H[Commit / Merge]
```

------------------------------------------------------------------------

# 3. Фазы разработки

## 🧱 Фаза 1 --- Базовая стабильность

-   Детерминированные команды:
    -   `покажи сценарий`
    -   `покажи сцены`
    -   `покажи раскадровки`
-   Полный trace layer
-   Git-интеграция (ветки + diff + commit)

## 🧠 Фаза 2 --- Строгий Planner + Loop

-   JSON-only planner
-   Execution loop с перепланированием
-   Self-fix workflow
-   Verification layer

## 🎨 Фаза 3 --- Asset Pipeline

-   Генерация ассетов
-   Загрузка из whitelist доменов
-   Трансформация
-   Unreal import helper

## 🧩 Фаза 4 --- Саморасширение

-   Автоматическое добавление новых модулей

# 6. Self‑Extension Workflow (Добавление новой функции)

``` mermaid
flowchart TD
    A[Команда: добавь функцию] --> B[Planner: анализ кода]
    B --> C[Создание feature-ветки]
    C --> D[Генерация новых модулей]
    D --> E[Регистрация в registry]
    E --> F[Тестирование]
    F -->|OK| G[Diff + Commit]
    F -->|FAIL| B
```

------------------------------------------------------------------------

# 7. Нефункциональные требования

-   Полная аудируемость
-   Невозможность silent-modifications
-   Все изменения через Git
-   Отсутствие выполнения shell-команд вне whitelist
-   Ограничение файловых операций workspace root

------------------------------------------------------------------------

# 8. Критерии Production‑готовности

Система считается production-ready если:

-   Нет "пустых" ответов без действий
-   Любая модификация имеет diff
-   Любое действие имеет trace
-   Ошибки автоматически исправляются через loop
-   Asset pipeline интегрирован с Unreal
